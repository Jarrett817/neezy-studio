数据存储划分规范：MD / SQLite 数据库 / 向量库
一、适合存 Markdown（MD）
原则：人可读、可手动编辑、原始存档、无需结构化查询、可直接拷贝迁移
AI 角色人设、性格设定、角色背景
所有系统提示词、Prompt 模板、指令规则
个人永久偏好、生活习惯、长期约定
手写笔记、日记、随笔、项目文档说明
本地知识库原始文档、教程、笔记原文
聊天完整会话原始记录归档备份
需要手动修改、版本留存、不依赖程序结构的纯文本
跨设备直接复制就能用的文本配置与记忆原文
二、适合存 普通 SQLite 数据库
原则：结构化字段、分页 / 排序 / 筛选、表关联、程序自动管理、业务配置
软件全局设置：主题、窗口布局、快捷键、外观配置
会话列表：会话标题、创建时间、更新时间、状态
标签、分类、文件夹、层级关联关系
模型配置、模型下载记录、收藏、浏览历史
本地用户资料、缓存、任务队列、同步状态
笔记 / 文档元数据：标题、时间、标签、归属、文件路径
应用运行日志、行为统计、权限配置、业务字典
三、适合存 向量库（SQLite + sqlite-vec）
原则：需要语义检索、模糊回忆、RAG 知识库、长记忆召回、按含义匹配
聊天记录切片（向量化片段）
跨天跨会话长期记忆片段
PDF/MD/ 本地文档 切片（私有 RAG 知识库）
项目知识点、碎片化知识、学习笔记切片
模糊语义搜索、不是关键词匹配的内容查找
上万条海量文本，无法全塞进上下文，需按需召回
智能联想、内容推荐、语义关联匹配业务
四、快速判定口诀
人能看懂、手动改、纯文本归档 → 存 MD
程序要列表、分页、关联、配置管理 → 存 SQLite
要模糊回忆、找相似、做 RAG 语义检索 → 存向量库
五、标准联动流程
原文本体 → 存 MD
元数据 / 结构化信息 → 存 SQLite
文本切片生成 Embedding → 存 向量库
用户提问 → 向量语义召回 → 调取 MD / 数据库原文 → 喂给大模型
