# 小红书博主 AI 运营工作台・最终端侧版产品文档

## 文档说明



* 适配技术栈：React 18 + React Router 7（Hash Router）+ Shadcn/UI + TailwindCSS + Tauri v2

* 运行环境：Windows 10+ /macOS 12+ 双端桌面客户端（家用 / 办公 PC 通用，8GB 内存即可流畅运行）

* 核心定位：**纯端侧 AI 驱动的私有化运营工具**，无云端依赖、无网络要求、无合规风险，所有 AI 能力（文案生成、截图提取、检索、记忆）均通过本地模型实现，打包后双击即用

* 核心模型：Qwen3-4B-Instruct（主模型）+ MiniCPM-V 2.0（图文 / OCR）+ bge-small-zh（检索）



***

## 一、产品核心定位

### 1.1 价值主张

为小红书个人博主提供「**零门槛、零成本、零风险**」的本地 AI 运营解决方案，无需 API Key、无需网络、无需专业设备，普通办公 / 家用 PC 即可流畅运行，通过端侧多模型协同，实现 “文案创作 + 截图导入 + 知识库沉淀 + 长期记忆” 全链路运营，沉淀私有化内容资产。

### 1.2 核心亮点（端侧专属优势）



1. **100% 本地运行**：无网络环境下正常使用所有功能，摆脱网络依赖，随时随地创作

2. **零成本使用**：内置端侧 AI 模型，无需 API Key、无调用次数限制、终身免费

3. **绝对安全合规**：数据（笔记、记忆、知识库）100% 本地存储，不经过任何云端服务器；截图提取仅本地处理，不涉及小红书接口 / 爬取，零封号风险

4. **低门槛适配**：8GB 内存办公 / 家用 PC 即可流畅运行，无需独显、无需 CUDA、无需复杂配置，双击安装即用

5. **轻量化打包**：软件安装包≤250MB（含核心模型），占用内存≤4.5GB，不卡顿、不拖慢电脑

6. **中文场景最优**：端侧模型针对中文 + 小红书场景深度优化，文案风格、截图识别准确率远超通用模型

7. **操作极简**：无需专业知识，截图导入、AI 生成、数据录入全流程 3 步搞定，上手即会

### 1.3 目标用户



* 小红书个人博主（美妆、穿搭、母婴、美食、知识、职场等垂直赛道）

* 办公 / 家用 PC 为主要设备，无专业 AI 硬件的普通创作者

* 注重数据隐私，拒绝云端存储的独立运营者

* 追求零成本 AI 工具，不想支付 API 费用的自媒体人

## 二、核心功能模块（端侧 AI 驱动）

### 2.1 工作台首页（每日运营中心）

#### 核心功能



1. **端侧 AI 每日简报**（本地定时生成，默认每日早 8 点，无需网络）

* 3 个定制选题推荐（Qwen3-4B 基于人设记忆 + 知识库自动生成）

* 今日最佳发布时段建议（基于本地历史发布数据统计）

* 热点关键词推荐（结合本地热点库 + 模型趋势分析）

* 待办清单（创作、复盘、知识库更新，支持 AI 自动添加）

1. **内容速览**

* 最近草稿、已发布笔记列表（本地 SQLite 存储）

* 核心数据概览（累计粉丝、笔记数、爆款数，手动录入后自动统计）

* 本周 / 本月内容进度（选题完成率、发布计划可视化）

1. **快捷入口**

* 新建笔记、截图导入、知识库管理、数据录入

* 模型设置（轻量 / 均衡模式切换）、数据备份 / 恢复

#### 设计要求



* 首页为默认启动页，信息层级清晰，突出 “每日选题” 和 “截图导入” 核心功能

* 显示当前模型运行状态（内存占用、模型版本），让用户直观了解性能

* 支持自定义每日简报生成时间、选题数量

### 2.2 智能创作中心（Qwen3-4B 核心驱动）

#### 核心功能



1. **全场景端侧 AI 生成**

* 标题生成 / 改写（支持 “爆款风、悬念风、干货风” 等小红书专属风格）

* 正文创作（基于人设记忆 + 知识库参考，支持分段生成、扩写、缩写，32K 上下文适配长文案）

* 标签推荐（结合本地标签库 + 历史偏好，生成精准标签组，可直接复制）

* 封面文案 / 配图思路（文字描述，适配小红书视觉风格）

1. **内容优化工具**

* 合规检测（端侧模型联动本地违禁词库，实时标记敏感词、限流风险词）

* 风格统一（一键校准文案语气，匹配账号人设，端侧实时处理≤500ms）

* 排版建议（小红书友好型分段、表情符号推荐、重点标注）

1. **草稿管理**

* 自动实时保存草稿（本地存储，防止意外丢失）

* 草稿分类（未完成、待发布、已发布）

* 历史版本回溯（保留最近 5 次修改记录，支持一键恢复）

#### 设计要求



* 创作界面分 “输入需求区 - AI 生成区 - 优化编辑区”，流程清晰，响应迅速

* 支持 “一键复制全内容”（标题 + 正文 + 标签），直接粘贴到小红书 APP

* 生成速度可视化（显示 “生成中…”，端侧推理延迟≤800ms）

### 2.3 截图导入功能（MiniCPM-V 2.0 核心驱动）

#### 核心功能（端侧 OCR + 图文理解，无需网络）



1. **多场景截图提取**

* 笔记文案提取：支持小红书 APP 截图（单篇笔记、多图笔记），自动提取标题、正文、标签

* 数据截图提取：支持小红书创作者中心数据截图，自动识别阅读量、点赞、收藏、评论数，结构化填充至数据复盘模块

* 对标内容提取：支持对标博主笔记截图，提取爆款文案、标题结构，自动存入知识库

1. **提取优化**

* 自动去重、格式整理（去除冗余水印、广告文字）

* 错误修正（端侧模型自动修正 OCR 识别错误，准确率≥98%）

* 一键导入：提取后可直接导入 “创作中心” 或 “知识库”，无需手动编辑

#### 设计要求



* 支持拖拽截图文件 / 粘贴截图到软件，触发自动提取

* 提取过程显示进度条，端侧处理延迟≤1 秒

* 提取结果支持手动编辑修正，确认后再导入

### 2.4 私有化知识库管理（bge-small-zh+SQLite-VSS 驱动）

#### 核心功能



1. **知识库分类管理**

* 预设分类：爆款案例、文案模板、赛道干货、违禁词库、对标博主、标签库

* 支持用户自定义分类（如 “节日选题库”“粉丝问答话术”）

1. **内容录入与编辑**

* 文本录入 + 截图导入（MiniCPM-V 2.0 提取后直接存入）

* 批量导入 / 导出（TXT/CSV 格式，支持备份迁移）

* 内容标签：给每条内容打标签，提升检索精准度

1. **端侧语义检索**

* 关键词检索 + 向量语义检索双模式（bge-small-zh 生成向量，SQLite-VSS 存储检索）

* 检索结果按 “相关性 + 使用频率” 排序，端侧检索延迟≤300ms

* 支持筛选分类、标签、创建时间

#### 设计要求



* 知识库列表支持分页、搜索、筛选，操作便捷

* 检索结果高亮显示匹配关键词，支持 “一键插入到创作区”

* 显示知识库总容量、向量索引状态，让用户了解存储情况

### 2.5 分层记忆系统（Qwen3-4B + 本地存储驱动）

#### 核心功能



1. **人设永久记忆**（用户手动配置，端侧模型实时调用）

* 账号定位（如 “平价美妆测评博主，面向学生党”）

* 语气风格（如 “活泼闺蜜风、干货严谨风、幽默吐槽风”）

* 排版习惯（如 “首段吸睛、分段≤3 行、结尾引导互动”）

* 内容禁忌（如 “不推荐医美、不使用绝对化用词”）

1. **内容长期记忆**（系统自动记录，端侧模型学习）

* 历史发布笔记（标题、正文、标签、发布时间、数据表现）

* 爆款 / 低效内容标记（用户手动标注，供模型优化生成方向）

* 常用标签组、固定句式、高频表情

1. **行为迭代记忆**（系统自动记录，实时优化模型输出）

* 用户对 AI 生成内容的修改、采纳、否决记录

* 高频使用的知识库内容、生成风格偏好

* 复盘时的优化建议采纳情况

#### 设计要求



* 记忆管理界面分 “人设配置”“内容记忆”“偏好记录” 三大板块，清晰可查

* 支持 “记忆权重调整”（如 “强化人设记忆，弱化某类内容记忆”）

* 支持 “记忆清理 / 重置”，保护用户隐私

### 2.6 数据复盘与规划中心

#### 核心功能



1. **数据录入与分析**

* 手动录入 + 截图提取双模式（MiniCPM-V 2.0 识别数据截图，自动填充）

* 数据可视化（柱状图 / 折线图展示单篇数据、周 / 月数据趋势，端侧本地渲染）

* 端侧 AI 复盘分析（Qwen3-4B 基于录入数据，分析 “标题 / 选题 / 标签” 优劣，给出优化建议）

1. **内容规划**

* 周 / 月选题排期（日历视图，支持添加选题、标记发布状态）

* 选题库管理（未使用、已使用、待优化选题分类）

* 创作节奏建议（基于历史发布频率，端侧模型推荐合理发布间隔）

1. **爆文归因拆解**

* 用户手动标记爆文后，Qwen3-4B 自动拆解 “标题关键词、正文结构、标签组合”

* 生成 “爆文模板”，自动存入知识库供后续复用

#### 设计要求



* 数据录入界面极简，支持快速填写（默认值、下拉选择）

* 可视化图表支持鼠标悬浮查看详情，支持导出图片

* 排期日历支持拖拽调整，关联草稿库

## 三、端侧模型架构（核心落地细节）

### 3.1 模型选型与参数（家用 / 办公 PC 适配）



| 模型名称              | 核心作用                | 量化版本      | 模型体积  | 内存占用   | 推理延迟   | 适配场景          |
| ----------------- | ------------------- | --------- | ----- | ------ | ------ | ------------- |
| Qwen3-4B-Instruct | 文案生成、选题、复盘、Agent 调度 | GGUF INT4 | 2.2GB | ≤2.5GB | ≤800ms | 核心主模型，覆盖文本类场景 |
| MiniCPM-V 2.0     | 截图 OCR、图文理解、数据提取    | GGUF INT4 | 1.2GB | ≤2GB   | ≤1 秒   | 截图导入核心模型      |
| bge-small-zh-v1.5 | 知识库语义检索、向量生成        | FP16      | 80MB  | ≤200MB | ≤300ms | 辅助检索模型        |

### 3.2 模型部署架构（Tauri+llama.cpp）

#### 部署核心原则



* 无需 Python、无需 CUDA、无需独显，核显 / 集显即可运行

* 模型预打包至软件安装包，用户无需额外下载

* 支持 “轻量 / 均衡” 双模式：8GB 内存默认轻量模式（优先保证流畅），16GB 内存自动切换均衡模式（提升生成质量）

#### 部署流程



1. 模型预处理：将原始模型转换为 llama.cpp 支持的 GGUF 格式（INT4 量化），减小体积

2. Tauri Rust 绑定：开发 tauri-plugin-llm 插件，封装模型加载、推理、销毁接口

3. 前端调用：通过`tauri.invoke`调用 Rust 接口，实现 “前端触发→端侧推理→结果返回” 闭环

4. 资源管理：模型文件存储在软件安装目录下，启动时自动加载，退出时释放内存

#### 目录架构（端侧模型专属）



```
src/

├── ...（原目录保留）

├── models/          # 模型配置、调用封装（前端层）

│   ├── llm.ts       # Qwen3-4B调用逻辑

│   ├── ocr.ts       # MiniCPM-V 2.0调用逻辑

│   ├── embedding.ts # bge-small-zh调用逻辑

src-tauri/

├── src/

│   ├── plugins/

│   │   ├── llm/            # Rust端模型推理插件（基于llama.cpp）

│   │   ├── ocr/            # Rust端OCR插件

│   ├── model\_manager/      # 模型加载、内存管理、模式切换

│   ├── handlers/           # 前端调用接口封装

├── resources/              # 预打包模型文件（GGUF格式）

│   ├── qwen3-4b-instruct.gguf

│   ├── minicpm-v-2.0.gguf

│   ├── bge-small-zh.gguf
```

## 五、部署与交付（端侧专属）

### 5.1 运行环境要求（家用 / 办公 PC 通用）



* Windows：Windows 10 及以上，64 位系统，≥8GB 内存，核显 / 集显即可

* macOS：macOS 12 及以上，Apple Silicon/Intel 芯片，≥8GB 内存

* 存储需求：安装包≤250MB，安装后占用空间≤5GB（含模型 + 数据）

* 网络需求：完全离线可用，仅首次安装时可选更新模型（非强制）

### 5.2 打包交付物



| 平台      | 交付格式              | 说明                                 |
| ------- | ----------------- | ---------------------------------- |
| Windows | .exe 安装包 + 绿色免安装包 | 一键安装，支持自定义安装路径，自动关联截图文件            |
| macOS   | .dmg 镜像安装包        | 拖拽安装，适配 Apple Silicon/Intel，支持托盘常驻 |

### 5.3 模型与数据管理



* 模型预打包：核心模型（Qwen3-4B+MiniCPM-V 2.0+bge-small-zh）预置于安装包，用户无需额外下载

* 数据备份：支持一键导出完整 SQLite 数据库文件（.db），包含所有笔记、知识库、记忆数据

* 模型更新：支持手动检查模型更新，下载增量更新包（≤500MB）

* 内存优化：闲置时自动释放模型内存，活跃时快速加载，不影响电脑正常使用
